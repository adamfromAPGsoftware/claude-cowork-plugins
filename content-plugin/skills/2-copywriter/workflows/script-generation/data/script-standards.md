# Script Standards

## Document Completeness

A complete script document must contain all of the following sections:

1. **Script Overview** — Concept, source brief, target length, date
2. **Direction & Angle** — Approved creative direction with title ideas and angle
3. **Scripted Intro** — Word-for-word teleprompter-ready intro with hook
4. **Body Segments** — Dot-point key talking points per segment (NOT word-for-word)
5. **CTA / Outro** — Call-to-action and closing
6. **YouTube Metadata** — Title options (3-5), description, tags
7. **Thumbnail Concepts** — Actionable visual concepts for Creative Director
8. **B-Roll Suggestions** — Practical B-roll ideas, especially for intro
9. **Teleprompter-Ready Section** — Clean formatted intro + key points for copy-paste

## Scripted Intro Standards

- Must have a strong hook in the first 5-10 seconds
- Written word-for-word, ready for teleprompter
- Should establish the video's value proposition early
- Tone should match brand voice
- Reference `creator-voice.md` from `inspiration/` for Adam's exact tone profile, contractions, and signature phrases

## Body Segment Standards

- Organized as dot-point talking points per segment
- Each segment should have a clear topic/theme
- Points should be concise but comprehensive enough to talk from naturally
- NOT word-for-word scripting — just key points

## YouTube Metadata Standards

- **Titles:** 3-5 options, SEO-conscious, under 60 characters preferred
- **Description:** Include key topics, timestamps placeholder, relevant links placeholder
- **Tags:** 10-15 relevant tags mixing broad and specific terms

## Teleprompter Section Standards (Borumi-Compatible)

- Output is for copy-pasting directly into Borumi for filming
- ALL CAPS section markers on their own line — each starts a new Borumi scene
- No line breaks within a section (Borumi treats newlines as new scenes)
- Intro presented word-for-word as a single continuous line
- Body points combined into one line per section, separated by " — "
- No markdown formatting

## Data-Driven Quality Standards

Based on empirical analysis of 15 short-form and 5 long-form high-performing reference videos.

### Hook Quality Criteria
- **Long-form hook:** Delivered in 3–7 seconds (~15–20 words), includes a specific number, uses a proven formula from the pattern library
- **Short-form hook:** Delivered in 2–3 seconds, pattern-interrupt or curiosity-driven
- **Every hook** MUST include a specific number (revenue, time, count) — never generic

### Pacing Standards
- **Short-form:** 3.3–4.0 words per second, 22–30 seconds optimal duration
- **Long-form:** 150–180 words per minute speaking pace

### Credibility Stacking Requirements (Long-Form)
- **One sentence only** — rattle off 2–3 stats in a single breath, no setup or preamble
- Timing window: ~5–8 seconds (literally one line, then move on)
- Example pattern: "I'm top 1% on Upwork, around $1M on the platform, and over $1M in revenue as an AI agency."
- Must include at least one specific revenue figure — never round to "six figures"
- No "let me explain why I'm qualified" setup lines — just state it and go
- Reference `creator-credentials.md` for Adam's actual proof points
- Reference `creator-voice.md` for credibility delivery style

### Long-Form Intro Structure Checklist
Every long-form scripted intro must feel like **natural conversation, not a template**. Target 35–45 seconds total, ~120 words. The sections below should FLOW into each other — no rigid blocks.

1. **Hook (0–5s):** Approved formula, under 20 words, specific number. Single breath.
2. **Credibility + Context (5–15s):** Single sentence rattling off 2–3 stats — no setup line, no backstory. Immediately connect to WHY this video matters ("and I use both every single day" or "and what I'm showing you today is the thing that replaced all of it"). ~10 seconds max, then MOVE ON.
3. **What You'll Learn (15–30s):** Natural tease of 2–3 things, woven conversationally. NOT "By the end of this video, you're going to..." followed by a bulleted list. Instead: "I'm going to show you [thing], and then [the surprising part nobody covers]." One flowing thought, not enumerated points.
4. **Bridge (30–40s):** Signature transition phrase — "with all the boring stuff out of the way", "without wasting any more of your time", or "so let's jump into it." One sentence max.

### Long-Form Intro Anti-Patterns (NEVER DO THESE)
- **"By the end of this video, you're going to..."** followed by a list — this is the #1 sign of a scripted intro. Instead, weave what they'll learn naturally.
- **"I'm going to... Then I'm going to... And then I'm going to..."** — repetitive "I'm going to" stacking sounds robotic. Use variety: "I'm going to show you X, and then Y — and here's the part nobody talks about."
- **Standalone "Barrier Removal" section** — "You don't need to be a developer. You don't need to know X. You don't need to understand Y." as its own paragraph is filler. If relevant, fold ONE sentence into the value tease naturally. Most videos don't need it at all.
- **Credibility as a paragraph** — if it takes more than one sentence to deliver your creds, you're over-explaining. Rattle them off and move.
- **Intros longer than 45 seconds** — if you haven't started the content by 45s, you've lost people. The real Adam videos average 35–40s before "let's jump into it."

### Short-Form CTA Standards
- Comment-gated: drive engagement via keyword CTA ("Comment [WORD] and I'll send you...")
- CTA as the first line of the caption
- Keep CTA under 10 words

### Visual Planning Standards
- 1 visual change per second during demo/tutorial segments
- Split-screen or visual proof in first 1–3 seconds
- Stage directions included for all visual elements (B-roll, screen recordings, overlays)
- Use motion graphic stage direction tags `[MG-A]` through `[MG-G]` (see `motion-graphic-types.md`) to specify visual support in scripts

### Visual Change Frequency (Long-Form)
- **Intro (first 60s):** Visual change every 4–8s (confirmed 6.3–8.6 changes/min across creators)
- **Agenda/walkthrough:** Visual change every 10–30s (smooth movements, not hard cuts)
- **Body:** Visual change every 15–45s
- Script should place `[MG-X]` stage directions at these intervals
- Rule: Never go more than 15s in the intro without a stage direction for a visual change

## Validation Criteria

- [ ] All 9 sections present and filled
- [ ] Intro is word-for-word and teleprompter-ready
- [ ] Body segments are dot-point format (not verbatim script)
- [ ] At least 3 title options provided
- [ ] Description includes structure for timestamps
- [ ] Tags are relevant and varied
- [ ] Thumbnail concepts are actionable
- [ ] B-roll suggestions are practical
- [ ] Teleprompter section is clean and copy-pasteable
- [ ] Content aligns with source concept brief
- [ ] Contractions used throughout (I'm, you're, don't — never formal forms)
- [ ] Signature phrases used where applicable ("jump into", "I'm going to show you")
- [ ] No generic AI openers ("hey guys", "what's up everyone")
- [ ] Tone matches calm authority profile (no clickbait, no hype language)
- [ ] "you" direct address appears every 2-3 sentences in intro
