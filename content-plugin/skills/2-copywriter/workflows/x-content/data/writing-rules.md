# X/Twitter Writing Rules

Format-specific writing constraints and rules for X content generation.

**Account type: X Premium** — 25,000 character limit applies. Do NOT default to 280-char threads for longer content. Use single long-form posts instead.

---

## Universal X Rules

- **Hook first:** First 140 characters are visible before "Show more" on mobile — the hook must land here regardless of post length
- **No hashtags** — X's algorithm no longer favours them; they add visual noise. None, ever.
- **No links in body** — Links are suppressed algorithmically even for Premium accounts. Put links in a reply/first comment instead.
- **No engagement bait that feels forced:** Organic CTAs only — avoid "like and RT if you agree" style
- **Line breaks for readability:** Short paragraphs separated by blank lines — X is a mobile-first feed
- **No filler words:** Cut ruthlessly. Confidence reads better on X.
- **Avoid LinkedIn phrasing:** "I'm excited to share", "Lessons learned", "Humbled by" — X culture rejects this
- **Authenticity over polish:** Raw, direct, slightly imperfect > corporate smooth
- **Text posts perform best for Premium accounts** — 0.9% median engagement vs. 0.4% for images. Lean text-first.

---

## Format Decision Tree (Premium account)

| Content type | Format | Target length |
|---|---|---|
| Quick take / contrarian opinion | Short post | 71–140 chars |
| Tip, insight, announcement | Short-medium post | 140–280 chars |
| YouTube repurpose / tutorial / analysis | Long-form single post | 500–2,500 chars |
| Deep dive / opinion piece | Long-form single post | 2,000–5,000 chars |
| Real-time commentary / news-jacking | Short post | 71–140 chars |

**Default for autopilot:** Long-form single post when repurposing YouTube content or writing a full technical/nurture post. Short post for quick contrarian takes.

**Threads are NO LONGER the default.** Long-form single posts accumulate all engagement in one place vs. fragmenting it across a chain where most readers drop off after post 3. Reserve threads only when the content is genuinely list-structured (e.g., "7 tools" where each item needs its own space).

## Short Post (71–280 chars) — Quick takes, hooks, opinions

**Sweet spot:** 71–140 chars for maximum engagement rate (retweets, replies).

**Structure:**
- Hook: One sharp sentence — a specific claim, number, or contrarian take
- No body needed if the hook is strong enough
- Optional: 1-line expansion or CTA if it fits within ~280 chars

**Rules:**
- Every word earns its place — if it doesn't add meaning, cut it
- Numbers and specifics stop the scroll — prefer "saved 4 hours" over "saved time"
- Full stops add weight — use sparingly for emphasis
- Present the character count in output

**Examples:**
```
5 Claude CoWork workflows. $2,800/month recovered. 1 hour of setup.

(89 chars)
```

```
Most businesses are spending their first 2 hours of every day finding information they could get automatically.

(111 chars)
```

## Long-Form Single Post (280–25,000 chars) — X Premium default for substantial content

**Target range:** 500–2,500 chars for most autopilot posts. 2,000–5,000 for genuine deep dives.

**Key rule:** The first 140 chars must function as a complete, standalone hook. Readers decide whether to tap "Show more" based on that alone.

**Structure:**
```
[Hook — 1–2 punchy lines, under 140 chars total. Must work standalone.]

[blank line]

[Context or problem setup — 2–4 lines]

[blank line]

[Core content — breakdown, story, or insight]
→ Use arrow lists for steps/workflows
→ Keep each line scannable

[blank line]

[Key takeaway — 1 quotable sentence]

[blank line]

[Optional soft CTA — 1 line. No links in body — put link in reply.]
```

**Rules:**
- First 140 chars = the "Show more" trigger — treat it like its own post
- Use blank lines between every section — X is mobile, dense walls of text lose readers
- Arrow (→) lists work for workflows and steps — same as LinkedIn
- No hashtags anywhere
- No links in body — state "link in reply" or just let the content stand alone
- Aim for a single clear argument arc: problem → insight → implication
- End before you've said everything — leave something for the replies

**Performance data:**
- Long-form posts vs. threads: 11,846 vs. 10,783 total impressions (Hootsuite experiment)
- All engagement concentrates on one post — no fragmentation across a chain
- Saves and bookmarks are higher for long-form — algorithm rewards saves

## Thread (use sparingly — only when content is genuinely list-structured)

**When to use:** Only when content is a numbered list where each item genuinely needs its own breathing room (e.g., "7 tools I use every day" with a paragraph per tool). If the content can fit in a long-form post, use that instead.

**Tweet sweet spot:** 5–8 posts. Under 5 = should be a long-form post. Over 10 = reader fatigue risk.

**Character target per tweet:** 150–220 chars. Posts above 240 chars drop to 44% read-through rate (vs. 68% for 150–220 chars).

**Tweet 1 (Hook):**
- Must work as a standalone post
- End with "Here's how:" or "A thread:" or "↓" to signal more
- Under 140 chars strongly preferred — this is the share unit

**Tweets 2–N:**
- Number explicitly: "1.", "2." or "1/7", "2/7"
- One idea per tweet — no cramming
- Each tweet must have standalone value

**Final tweet:**
- Summarise in 1–2 lines
- "Follow @{YOUR_HANDLE_PERSONAL}_ for more on [topic]"

**Rules:**
- No hashtags
- No body links — put link in final tweet or a reply
- If it can fit in a long-form post, use long-form instead

## Image Post

**Caption rules:**
- Caption amplifies the image — does NOT describe or duplicate what the image shows
- 1–3 lines: hook + brief expansion
- Character budget: aim for 140–280 chars (short, punchy captions work better with images)
- No hashtags

**Image specs:**
- Recommended: 1200×675 (16:9 landscape) or 1080×1080 (square)
- Max file size: 5MB (JPEG/PNG)

---

## Video Post

**Caption rules:**
- First line = visual promise: tell the viewer what they're about to see
- 1–3 lines total
- Reference what makes the video worth watching
- Character budget: 140–280 chars preferred for video captions

**Video specs (X Premium):**
- Max length: 2 hours (Premium)
- Max file size: 512MB+
- Supported formats: MP4, MOV
- Recommended: 1080p, 16:9 or 1:1

**Rules:**
- Autoplay is silent — first 3 seconds must be visually compelling without audio
- "Watch me X in Y minutes" is a proven caption format for tutorial/demo content
