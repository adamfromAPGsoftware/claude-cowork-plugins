# X/Twitter Writing Rules

Format-specific writing constraints and rules for X content generation.

---

## Universal X Rules

- **Hook first:** The first 140 characters are visible without "Show more" on mobile — the hook must land here
- **No hashtags in body** (or max 1 at end, rarely): X's algorithm no longer favours hashtags; they add visual noise
- **No engagement bait that feels forced:** Organic CTAs only — avoid "like and RT if you agree" style
- **Line breaks for readability:** Short paragraphs separated by blank lines — X is a mobile-first feed
- **No filler words:** "I think that maybe possibly..." — cut ruthlessly. Confidence reads better on X
- **Avoid LinkedIn phrasing:** "I'm excited to share", "Lessons learned", "Humbled by" — X culture rejects this
- **Authenticity over polish:** Raw, direct, slightly imperfect > corporate smooth

---

## Single Post (≤280 chars)

**Character budget:** 280 chars maximum (URLs count as 23 chars regardless of length)

**Structure:**
- Hook: First sentence, must be punchy and standalone
- Body: Maximum 1–2 additional sentences
- CTA: Optional — only if it fits naturally within budget

**Rules:**
- Every word must earn its place — if it doesn't add meaning, cut it
- Numbers and specifics stop the scroll — prefer "saved 4 hours" over "saved time"
- Full stops add weight — use sparingly for emphasis
- No mid-post hashtags; one at the very end only if highly relevant
- Present the post character count in the output

**Sweet spot format:**
```
[Hook sentence — punchy, specific]

[1-2 sentences of expansion or proof]

[Optional: 1-line CTA]
```

## Thread (2–20 tweets, 280 chars per tweet)

**Thread sweet spot:** 5–12 tweets. Under 5 = should be a single post. Over 12 = reader fatigue risk.

**Tweet 1 (Hook tweet):**
- Must work as a standalone post — if the reader doesn't engage here, they won't read the thread
- End with 🧵 (thread emoji) or "Here's how:" or "A thread:" to signal more
- Optionally include "Save this" or "Bookmark this" as a retention prompt
- Do NOT start the first tweet with "Thread:" — that's dead text

**Tweets 2–N (Body tweets):**
- One idea per tweet — no cramming
- Number them explicitly: "1.", "2." etc. for listicle threads
- Each tweet must have standalone value even out of context
- Use line breaks within tweets to aid readability

**Final tweet (CTA tweet):**
- Summarise the thread in 1-2 lines
- Add a follow CTA: "Follow @[handle] for more [topic]"
- Add an engagement or bookmark hook: "RT if this helped" or "Save this thread"
- Do NOT end abruptly — the final tweet is premium CTA real estate

**Rules:**
- Total thread character budget = 280 × number of tweets
- Number continuity: if a tweet is numbered, keep numbering consistent
- Bookmark hook can appear in tweet 1 or 2 (not just the final tweet)

## Long Post (≤25,000 chars — X Premium+ only)

**Format:** Article-style with markdown-style headers (X renders some formatting natively)

**Structure:**
- Opening paragraph: Standalone hook — must work as a scroll-stopper in the feed
- Headers: Use bold text or ALL CAPS to create section headers for skimmability
- Body sections: 3–5 paragraphs each, with line breaks between paragraphs
- Closer: Strong final section with clear takeaway + follow CTA

**Rules:**
- First paragraph = above-the-fold. Must be gripping.
- Use "---" or white space to separate major sections
- Bullet points work for lists — keep each bullet punchy
- Longer ≠ better: target 2,000–5,000 chars for most long posts; 5,000+ only for genuine deep dives
- Always include a clear argument arc: problem → insight → solution/implication

## Image Post

**Caption rules:**
- Caption amplifies the image — it does NOT describe or duplicate what the image shows
- 1–3 lines: hook + brief expansion
- The image should be self-explanatory; caption adds context or POV
- Character budget: aim for ≤280 but can extend if the image needs context
- No mid-post hashtags

**Image specs:**
- Recommended: 1200×675 (16:9 landscape) or 1080×1080 (square)
- Max file size: 5MB (JPEG/PNG), 15MB (GIF)
- Text-heavy images reduce reach — prefer visuals with minimal text

**Rules:**
- If the image is data/stats: caption gives the "so what" not the data itself
- If the image is a diagram: caption sets the context
- If the image is a photo/screenshot: caption provides the reaction or framing

---

## Video Post

**Caption rules:**
- First line = visual promise: tell the viewer exactly what they're about to see
- 1–3 lines total in caption
- Reference what makes the video worth watching (the demo, result, or reveal)
- Character budget: ≤280 preferred for video captions

**Video specs (X Premium):**
- Max length: 2 hours (Premium), 2:20 minutes (standard)
- Max file size: 512MB (standard), higher with Premium
- Supported formats: MP4, MOV
- Recommended: 1080p, 16:9 or 1:1 aspect ratio

**Rules:**
- Autoplay is silent — first 3 seconds must be visually compelling without audio
- Caption sets expectation; video must deliver on that promise
- "Watch me X in Y minutes" is a proven caption format for tutorial/demo content
