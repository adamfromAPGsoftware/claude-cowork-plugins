### What the Registration Covers

Registration is for **R&D activities**, not expenditure. You describe:
- What R&D activities were conducted
- The core R&D activity descriptions (hypothesis, uncertainty, methodology)
- Supporting R&D activity descriptions and their relationship to core activities
- Whether any activities were conducted overseas

## 3. Eligible R&D Activities

### Core R&D Activities

Must satisfy ALL of:
1. **Experimental activities** — systematic progression of work based on principles of established science
2. **Purpose** — generating new knowledge
3. **Technical uncertainty** — outcome cannot be known or determined in advance
4. **Systematic investigation** — planned investigation following hypothesis, test, evaluate methodology

### Supporting R&D Activities

Activities directly related to core R&D activities:
- Must be **directly related** to a registered core R&D activity
- Must be undertaken for the **dominant purpose** of supporting the core activity

### Software-Specific Guidance

**Eligible (involves technical uncertainty):**
- Developing new algorithms or AI models where outcome is genuinely uncertain
- Building novel software architectures not previously achieved
- Experimental development to resolve technical challenges
- AI agent orchestration with uncertainty in coordination approaches

**NOT Eligible (routine/predictable):**
- Standard website development or app builds
- Bug fixes, maintenance, patches
- Adapting existing solutions to new clients
- Using off-the-shelf tools in standard configurations

## 4. Overseas Activities

### General Rule

Core R&D activities must typically be conducted in your jurisdiction. Check your local rules for overseas activity requirements.

### Overseas Finding Requirement

To claim expenditure on overseas R&D activities, you may need prior approval. Key considerations:
- Activity qualifies as core or supporting R&D
- There is a related core R&D activity conducted domestically
- The overseas activity cannot be conducted domestically
- Check deadline requirements — these are often strict with no extensions

## 5. Eligible Expenditure

### Types of Claimable Expenditure

| Type | Description | Examples |
|------|-------------|---------|
| Contractor payments | Payments for R&D services | Freelancers, specialist developers |
| Salary & wages | Employee/director time on R&D | R&D-related salary (apportioned) |
| Software & tools | Development tools used for R&D | AI APIs, IDEs, hosting |
| Cloud hosting | Infrastructure for R&D projects | Hosting platforms (apportioned) |
| Depreciation | Assets used for R&D | Hardware (apportioned) |

### Apportionment

When expenditure is partially R&D and partially non-R&D:
- Must use a reasonable and defensible apportionment methodology
- Document the basis for apportionment
- Common methods: time-based, project-based
- Be conservative

## 6. Documentation Requirements

### Contemporaneous Records

Records must be created at or around the time R&D activities were conducted.

### What to Maintain

| Record Type | Purpose | Sources |
|-------------|---------|---------|
| Activity descriptions | Define what R&D was done | Skill definitions, pipeline docs |
| Technical decisions | Show uncertainty and methodology | Git commits, PR descriptions |
| Time records | Apportion effort | Time tracking, milestone records |
| Financial records | Prove expenditure amounts | Bank transactions, invoices |
| Outcome records | Show what was learned | Code outputs, test results |
