# Generate Close Page

## Purpose

After the discovery call, analyse the transcript via 4-pass extraction and update the existing discovery page with real findings. The hypothesis pain points become evidence-based. The "You Today" column in the competition table gets filled with actual client data. The waste section uses real numbers from the call. The result is a personalised close page ready to send.

## Prerequisites

- A discovery page should already exist at `clients/{slug}/close-page/close-page.html` (generated by GDP)
- If no discovery page exists, warn the user and offer to run GDP first, or proceed with a fresh standard/urgency template instead

## Process

### Step 1: Confirm Transcript

Client slug is already set from activation (`{client_slug}`).

1. **List transcripts** — Show available files in `clients/{client_slug}/meetings/` and auto-select the most recent one (by date in folder name)
2. **Confirm:** `Found transcript: clients/{slug}/meetings/{YYYY-MM-DD}-{title}/transcript.txt — Using this one? [y/n]`
3. **Industry tag** — Confirm or ask (e.g. `trades`, `ndis`, `home-services`, `real-estate`, `construction`)

Read the transcript file.

### Step 2: Four-Pass Extraction

Run four focused passes through the transcript. Each pass has a single lens — do not blend lenses across passes. Treat each pass as a fresh read.

---

**Pass 1 — Process Flow & Pain Points**

Extract:
- Business lifecycle stages visible (Acquisition → Quoting → Onboarding → Fulfilment → Retention)
- Key processes described (how they currently do things step by step)
- Explicit pain points (things said to be broken, slow, manual, frustrating)
- Time estimates mentioned (hours/week, hours/day, frequency of tasks)
- Headcount mentioned per role or function

For each item, capture:
- `speaker` — who said it (Adam vs. client, use name if identifiable)
- `quote` — verbatim words, not paraphrased. **Clean up any swear words** — replace with a neutral equivalent that preserves the meaning. Quotes should read naturally and be presentable in client-facing assets.
- `confidence` — HIGH (explicitly stated with specifics) | MEDIUM (implied/inferred) | LOW (assumed, needs confirmation)
- `timestamp` — if visible in transcript

---

**Pass 2 — Tools & Tech Stack**

Extract every software tool, app, or platform mentioned. For each:
- `tool_name`
- `current_plan` — stated plan/tier if mentioned
- `seats` — number of users if mentioned
- `use_case` — how it's used in their process
- `workarounds` — manual steps they do that the tool should handle
- `quote` — verbatim mention
- `confidence`

---

**Pass 3 — Time, Waste & Labour Rates**

Extract every quantifiable waste signal AND any data about what staff cost:
- Hours/week spent on manual tasks
- Frequency of repetitive activities
- Volume figures (number of leads, clients, jobs, invoices handled)
- Dollar figures mentioned (revenue, costs, losses)
- Any "we lose X" or "it takes Y hours" type statements
- **Staff costs** — any mention of hourly rates, salaries, wages, or what roles earn

For each waste signal, capture the standard fields plus:
- `hours_per_week` — if stated or calculable
- `headcount_affected` — how many staff do this
- `waste_type` — `manual_data_entry` | `duplicate_work` | `no_followup` | `communication_gap` | `missing_automation`

**Determine the blended hourly rate for this client:**

From any salary/wage/rate data extracted, calculate a blended hourly rate:
- Salary → divide by 2,080 to get hourly (e.g., $55k ÷ 2,080 = $26.44/hr)
- If multiple roles involved, average them weighted by headcount
- If nothing stated: use $50/hr as fallback, flag as `confidence: LOW`
- Store as `blended_hourly_rate_aud` and `blended_rate_confidence` in the audit data-lite

After extraction, for each HIGH or MEDIUM confidence waste item with calculable hours, run the waste calculator:

```bash
python3 scripts/waste-calculator.py --help

python3 scripts/waste-calculator.py \
  --hours-per-week 5 \
  --headcount 3 \
  --hourly-rate {blended_hourly_rate_aud} \
  --activity "manual invoice data entry"
```

Store the `annual_waste_aud` result back into the waste item.

---

**Pass 4 — Decision Points & Close Intelligence**

Extract:
- Conditional logic: "if X happens, we do Y" — decision nodes
- Decision-making bottlenecks described (who has to approve what)
- Objections or hesitations surfaced during the call
- Moments of positive engagement — tone shifts, "yes exactly", enthusiasm
- The single most impactful waste or pain point — the one they lingered on, repeated, or showed the strongest reaction to
- **Competition table data** — anything the client said about their own operations that maps to competition metrics (lead response time, quote turnaround, onboarding, follow-up, reporting, cross-sell)

---

### Step 3: Identify Top Waste Item

From all waste items extracted, select the single strongest candidate for the close page hero:
- Highest confidence (prefer HIGH)
- Most quantifiable (has annual_waste_aud calculated)
- Most emotionally resonant (they reacted to it or brought it up themselves)

This becomes `top_waste_item`.

### Step 4: Map "You Today" Column

From the transcript findings, map what the client said about their operations to the competition metrics:

| Metric | What to extract from transcript |
|--------|-------------------------------|
| Lead response time | How fast they respond to enquiries, after-hours coverage |
| Quote turnaround | How long quotes take, manual vs automated |
| Client onboarding | Intake process, manual steps, confirmation methods |
| Follow-up system | How they chase leads, collect reviews, re-engage |
| Data and reporting | CRM usage, spreadsheets, visibility into operations |
| Cross-sell and upsell | Whether they proactively sell additional services |
| Industry-specific | Whatever was set in competitor-research.json |

Store as `you_today` entries — these replace "Discovered on call" in the competition table.

### Step 5: Compile audit-data-lite

Update the existing `audit-data-lite.json` with real data from the transcript, replacing hypothesis entries:

```json
{
  "client_slug": "",
  "company_name": "",
  "industry_tag": "",
  "discovery_date": "",
  "transcript_source": "",
  "contact": {
    "name": "",
    "role": "",
    "company_size": ""
  },
  "blended_hourly_rate_aud": null,
  "blended_rate_confidence": "HIGH|MEDIUM|LOW",
  "blended_rate_source": "",
  "business_stages_covered": [],
  "tools": [],
  "pain_points": [],
  "waste_items": [],
  "top_waste_item": {},
  "you_today": {
    "lead_response_time": "",
    "quote_turnaround": "",
    "client_onboarding": "",
    "follow_up_system": "",
    "data_and_reporting": "",
    "cross_sell_upsell": "",
    "industry_specific": ""
  },
  "objections": [],
  "positive_signals": [],
  "data_gaps": []
}
```

Validate:
```bash
python3 scripts/validate-audit-data-lite.py --file clients/{slug}/audit/audit-data-lite.json --verbose
```

Fix any critical or high severity findings before proceeding.

```
═══════════════════════════════════════════════════

   Calculation: {hrs}/wk × {headcount} staff × ${rate}/hr × 52 = ${annual}/yr (${monthly}/mo)

   Lead response: {what they said}
   Quote turnaround: {what they said}
   Client onboarding: {what they said}
   Follow-up: {what they said}

═══════════════════════════════════════════════════

═══════════════════════════════════════════════════

 Which template?

   Full price: ${audit_value}

```

Ask: **"Here's what I extracted. Any corrections before I update the page?"**

Wait for explicit confirmation.

### Step 7: Update Close Page HTML

Run the generator in update mode:

```bash
# Standard update
python3 scripts/generate-close-page.py \
  --client-slug {client_slug} \
  --mode update \
  --template standard \
  --booking-url "{booking_url}"

# Urgency update
python3 scripts/generate-close-page.py \
  --client-slug {client_slug} \
  --mode update \
  --template urgency \
  --booking-url "{booking_url}" \
  --audit-value "{audit_value}" \
  --discounted-rate "{discounted_rate}" \
  --discovery-datetime "{discovery_datetime_iso}"
```

The script:
1. Reads the existing discovery page HTML
2. Replaces hypothesis pain points with real quoted pain points
3. Fills "You Today" cells with real findings
4. Updates waste numbers with actual calculated values
5. Adds client quotes throughout
6. Updates hero subheadline with a specific reference to the call

If no discovery page exists, generates from scratch using the standard or urgency template.

**Review the output** — verify:
- Pain points now have real quotes, not hypotheses
- "You Today" column filled with specific findings
- Waste numbers match audit-data-lite
- Hero subheadline references something specific from the call

**Anti-patterns — enforce hard:**
- Zero generic sentences. Every sentence references this client specifically.
- No "many businesses struggle with..." — banned.
- Every number traces to the confirmed audit-data-lite.
- Case study and credibility sections are static — never modified.

### Step 8: Deploy

```bash
bash scripts/deploy.sh
```

Report:
```
CLOSE PAGE UPDATED & DEPLOYED — {company_name}
URL: {close_url from clients/clients.json}
Secret link — ready to send.
```

## CRM Lead Update

After deploying the close page, if `crm.lead_id` is set:

1. `get_lead(lead_id)` → check current stage
2. If stage is in {"Discovery Call - Scheduled", "Contacted"}: `update_lead(lead_id, stage: "Discovery Call - Completed")`
3. If stage is manual: add lead comment instead, do NOT override stage
4. If deal value was calculated from waste: `update_lead(lead_id, value: {annual_waste_aud})`
5. `create_lead_comment(lead_id, content)`:
   ```
   Discovery call completed. Close page updated with real findings.
   Waste items: {count}. Annual waste: ${total}/yr.
   Key pain points: {top 3 pain points}.
   ```
6. Best-effort — if CRM calls fail, log warning and continue.

Then ask: "Generate the follow-up email and SMS? [GFE]"
