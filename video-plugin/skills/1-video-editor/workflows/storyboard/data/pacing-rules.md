# Pacing Rules

## Visual Event Targets by Section Type

| Section Type | Target (events/min) | Minimum | Notes |
|-------------|---------------------|---------|-------|
| Hook | 15+ | 15 | Maximum visual energy — every second should have a change |
| Intro | 12-15 | 12 | High energy to establish topic and retain viewer |
| Body | 7-10 (short-form) / 4-8 (long-form) | 4 | Sustainable pace — long-form uses lower density per inspiration analysis |
| Chapter Transition | N/A | — | 2-3 second cards, not rated for pacing |
| CTA | 12-15 | 12 | Match intro energy for final push |

## What Counts as a Visual Event

Each of these counts as ONE visual event:

1. **Segment change** — Transition from one visual type to another (e.g., speaker → B-roll)
2. **Caption appearance** — New text appearing on screen
3. **Caption disappearance** — Text leaving the screen
4. **B-roll cut in** — B-roll overlay starting
5. **B-roll cut out** — B-roll overlay ending
6. **Motion graphic appearance** — MG clip starting
7. **Template transition** — Change in active Remotion template
8. **Zoom change** — SubtleZoom starting or ending
9. **PowerCaption burst appearance** — Each burst appearing on screen counts as 1 event
10. **PowerCaption word reveal** — Each individual word snapping in within a burst counts as 1 event
11. **SocialProofStack item** — Each data item appearing

## Evaluation Rules

### PASS
Events/min is within the target range for the section type.

### WARN
Events/min is within 2 of the target range boundary.
- Example: Body section at 6 events/min (target 7-10) → WARN (1 below minimum)
- Example: Intro section at 11 events/min (target 12-15) → WARN (1 below minimum)

### FAIL
Events/min is more than 2 outside the target range.
- Example: Hook at 10 events/min (target 15+) → FAIL (5 below minimum)
- Example: Body at 4 events/min (target 7-10) → FAIL (3 below minimum)

## Remediation Strategies

### Pacing Too Slow (below target)

1. **Add PowerCaption bursts** — Add more keyword burst moments per section (adds burst + word events)
2. **Insert B-roll cuts** — Break up long speaker segments with screen recordings or MG
3. **Add SubtleZoom** — On key statements (1 event per zoom)
4. **Split segments** — Break a 10s segment into two 5s segments with different visual treatments
5. **Add SocialProofStack** — Insert data/testimonial cards at natural pauses

### Pacing Too Fast (above target)

1. **Reduce PowerCaption burst count** — Fewer bursts per section (fewer word reveal events)
2. **Extend B-roll segments** — Hold B-roll longer instead of cutting quickly
3. **Remove redundant transitions** — Eliminate unnecessary visual changes
4. **Use fewer words per burst** — Shorter bursts generate fewer word reveal events

## Section Duration Guidelines

| Section | Typical Duration | Notes |
|---------|-----------------|-------|
| Hook | 3-8 seconds | First impression — must be compelling |
| Intro | 30-90 seconds | Establish topic, build curiosity |
| Body (per chapter) | 2-8 minutes | Main content delivery |
| Chapter Transition | 2-3 seconds | Visual separator between chapters |
| CTA | 15-45 seconds | Call to action before end |

## PowerCaption Density Calculation

PowerCaption generates events from both burst appearances and word reveals:
- A 3-word burst = 1 (burst appear) + 3 (word reveals) = 4 events in ~2s ≈ 120 events/min if used continuously
- Typical burst density: 1 burst per 3–5 seconds = 6–16 events/min (suitable for all section types)
- Reserve 2–3 bursts per section; target keyword phrases, not every word of dialogue
- Each burst should be 2–4 ALL CAPS words with one orange highlight word

## Long-Form Pacing Rules

These rules apply specifically to long-form tutorial content (16:9, 5–60+ min).

### Cut Density by Section (Confirmed from Inspiration Analysis)

| Section | Cuts/Min | Visual Change Frequency |
|---------|----------|------------------------|
| Hook (first 15s) | 12–17.5 | Every 2–4s |
| Value prop / authority | 5–12 | Every 5–8s |
| Agenda walkthrough | 0–5 | Every 10–30s (smooth movements instead of cuts) |
| Body (screen share) | 0–2 | Layout changes every 30–90s |

### What Counts as a Cut vs Visual Event

- **Cut**: A layout change — switching between segment types (speaker → B-roll, speaker → screen share, etc.)
- **Visual event**: Any visual change including captions, zoom changes, reveals, PiP toggling — broader than cuts
- Body target of 7–10 events/min is achieved through 0–2 cuts/min + visual variety (captions, zooms, reveals) bringing total to ~4–8 events/min

### Updated Body Target

For long-form content, the body pacing target is adjusted:
- **Body:** 4–8 events/min (lowered from 7–10 to match observed long-form patterns)
- Short-form body retains the original 7–10 target

### Hold Duration Rules

- **Max speaker-only hold:** 15s (then cut to B-roll, overlay, or screen share)
- **Max screen share hold without visual change:** 30s (then zoom, pan, or return to speaker)
- **Speaker return cadence:** Every 45s–2 min during screen share sections
- **Title card duration:** 2–4s
- **B-roll clip duration:** 3–6s
- **WhiteFlash transition:** 18 frames max (0.3s at 60fps / 0.6s at 30fps) — quick, snappy flash bridging intro-to-body cuts. Never hold longer than 18 frames.
