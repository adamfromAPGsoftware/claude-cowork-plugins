## Section 10: Remotion Implementation Notes

To replicate this style in Remotion, focus on the following technical implementations:

- **No PiP Component Needed:** Do not build a PiP overlay for the screen share scenes.
- **The UI Highlight Box Component:**
  - Create a reusable `<HighlightBox />` component.
  - **Props:** `x`, `y`, `width`, `height`, `opacity`.
  - **CSS:**
    ```css
    border: 3px solid white;
    borderRadius: 8px;
    boxShadow: 0 10px 25px rgba(0,0,0,0.2), 0 0 15px rgba(255,255,255,0.5);
    backgroundColor: transparent; /* Or a very faint white with backdrop-filter: brightness(1.1) */
    position: absolute;
    ```
  - **Animation:** Use `spring()` to animate the `opacity` from 0 to 1, and slightly scale it from `1.05` to `1.0` for a "snapping" effect.
- **Screen Share Zooming:**
  - Wrap the screen recording in a container with `overflow: hidden`.
  - Use `interpolate()` tied to `useCurrentFrame()` to animate the `transform: scale()` and `transformOrigin` properties.
  - Example: `transformOrigin: '20% 80%'` to zoom into the bottom-left terminal window.
- **Side-Graphic Layout (Theory Section):**
  - Create a composition with a flex container.
  - Left div: `width: 45%`, `padding: 40px`, contains the animated `<Card />` components.
  - Right div: `width: 55%`, contains the speaker video.
  - Apply a slight `translateX` to the speaker video to ensure they are centered within their new 55% bounding box, rather than cut off.
- **Lower Third Component:**
  - `position: absolute`, `bottom: 10%`, `left: 50%`, `transform: translateX(-50%)`.
  - Background: `#E0F2FE`.
  - Border-top: `4px solid #F97316`.
  - Use `spring()` for a slide-up-and-fade-in entrance.
