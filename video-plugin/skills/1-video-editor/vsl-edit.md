# [VE] VSL Edit

**Goal:** Take recorded VSL footage (single talking-head recording) and produce a fully edited Video Sales Letter with emphasis text overlays, icon pop-ups, B-roll cuts, full-screen graphics, and aggressive subtitles — rendered via Remotion.

**Role:** VSL production specialist. Combines direct-to-camera footage with motion graphics, B-roll, and text overlays using proven direct-response video editing patterns.

---

## Drop and Go

Put your recorded VSL video in `{project_folder}/{project-slug}/video-editor/video-ingest/` along with the VSL edit instructions file (generated by the marketing plugin's VSL Generator [GE] capability). Run [VE]. The pipeline auto-detects both files and proceeds.

---

## Key Difference from [LF] Long-Form Edit

| Aspect | LF (Tutorial) | VE (VSL) |
|--------|---------------|----------|
| Structure | Intro/body split | Single continuous video |
| Pacing | 4-8 events/min body | 20-25 layout changes/min throughout |
| Zoom | Jump-cut zooms on speaker | No zoom — static framing, energy from cuts |
| Graphics | Educational MGs, chapter cards | Proof graphics, value stacks, emphasis text |
| B-roll | Tutorial-relevant clips | B&W desaturated "pain state" footage |
| Subtitles | Phrase bursts, lower-third | Aggressive subtitles with green highlights + large breakout text |
| MG density | 4+ intro, light body | Rapid — overlay every 3-5 seconds |

## Pipeline Steps

### Step 1: Init
- Auto-detect VSL video in drop folder
- Auto-detect edit instructions file (vsl-edit-instructions.md)
- Probe video metadata, create registry YAML, generate 720p proxy
- Load edit instructions: section map, MG insertion points, asset checklist

### Step 2: Analysis
- Audio analysis (AA) for speech regions and silence detection
- Transcription (TR) via DeepGram Nova-3 with word-level timestamps
- No visual analysis needed (VSL is fully scripted — section boundaries come from edit instructions)
- No boundary detection needed (no intro/body split)

### Step 3: Clipping
- Dead air removal only — clip silence/filler using audio + transcript signals
- Single output clip (no intro/body split)
- Remap transcript to clipped timeline
- Extract audio for Remotion

### Step 4: Storyboard
- Load edit instructions for section map and MG insertion points
- Map script sections to clipped video timestamps using transcript alignment
- Plan overlay timeline:
  - **Subtitles:** 100% coverage, phrase-level bursts, green (#72E032) highlights on emphasis words
  - **Emphasis text breakouts:** Large text beside speaker at thesis statements (from edit instructions)
  - **Icon pop-ups:** Spring-animated logos at tool/brand mentions (from transcript)
  - **B-roll cuts:** B&W desaturated footage at pain/story sections (from edit instructions)
  - **Full-screen graphics:** UI mockups, value stacks, guarantee badges (from edit instructions + Hera MGs)
- Validate against pacing rules (V1–V12)
- Validate against compliance checklist

### Step 5: Assets
- Generate Hera motion graphics from edit instructions MG specs
- Source B-roll footage (stock or provided)
- Apply B&W treatment to B-roll: `grayscale(100%) contrast(120%)`
- Collect icon assets for pop-ups
- Extract audio from clipped video

### Step 6: Remotion Edit
Follow the Remotion architecture from compliance checklist (AR1–AR5):

**Layer stack (bottom to top):**
1. `<OffthreadVideo>` — single continuous speaker passthrough (muted)
2. `<Audio>` — single continuous audio track
3. B-roll overlay `<Sequence>` components (with grayscale filter)
4. Full-screen graphic overlay `<Sequence>` components
5. Icon pop-up `<Sequence>` components (spring animation)
6. Emphasis text `<Sequence>` components
7. Subtitle `<Sequence>` components (with green highlights)

**Component library:**
- `<DynamicSubtitles>` — phrase-level with `highlightWords` array → green (#72E032)
- `<EmphasisText>` — large bold text, hard-cut in/out, white/green mix
- `<SpringIcon>` — tool logos with `spring({ damping: 12, stiffness: 100 })`
- `<BRollOverlay>` — `<OffthreadVideo>` with `filter: grayscale(100%) contrast(120%)`
- `<FullScreenGraphic>` — dark background overlay with content (MG video or static)

**QA checklist:**
- All pacing rules V1–V12 pass
- All compliance checklist items pass
- Audio continuous across all visual transitions
- Subtitles cover 100% of speaker dialogue
- At least 1 emphasis text breakout every 15-20s of speaker time
- All tool mentions have corresponding icon pop-ups
- CTA visual present in final 15s

### Step 7: Render
- `npx remotion render` to ProRes or H.264 MP4
- Verify output duration matches expected runtime
- Verify audio/video sync
